# 🎨 Diseño de Interfaz (UI/UX)

> Documento que define la experiencia de usuario, el flujo de navegación y los principios de diseño de la aplicación.

---

# 📖 Introducción

La interfaz de usuario será uno de los pilares fundamentales del proyecto.

El objetivo principal es que cualquier usuario pueda añadir un producto a la lista en menos de cinco segundos, minimizando el número de pulsaciones y eliminando cualquier elemento innecesario.

La aplicación estará diseñada siguiendo una filosofía **Mobile First**, ya que el principal dispositivo de uso será el teléfono móvil.

---

# 🎯 Objetivos de diseño

La interfaz deberá cumplir los siguientes objetivos:

- Ser intuitiva.
- Ser rápida.
- Requerir el mínimo número de acciones posibles.
- Funcionar perfectamente en dispositivos móviles.
- Adaptarse correctamente a escritorio.
- Tener una apariencia moderna y limpia.

---

# 📱 Filosofía Mobile First

Toda la interfaz será diseñada inicialmente para pantallas de móvil.

Posteriormente se adaptará para tablet y escritorio mediante diseño responsive.

---

# 🎨 Principios de diseño

## Simplicidad

La aplicación únicamente mostrará la información necesaria.

Se evitarán:

- Menús innecesarios.
- Pantallas intermedias.
- Formularios largos.
- Opciones poco utilizadas.

---

## Rapidez

El usuario deberá poder:

- Abrir la aplicación.
- Añadir un producto.

En menos de cinco segundos.

---

## Consistencia

Todos los componentes compartirán:

- Mismo estilo.
- Mismos colores.
- Mismos tamaños.
- Mismo comportamiento.

---

## Accesibilidad

Se utilizarán:

- Botones grandes.
- Tipografía legible.
- Buen contraste.
- Iconografía clara.

---

# 🧭 Flujo de navegación

## Primera apertura

```text
Inicio
   │
   ▼
Introducir código de vivienda
   │
   ▼
Guardar código localmente
   │
   ▼
Lista principal
```

---

## Uso habitual

```text
Escanear NFC
      │
      ▼
Lista principal
```

El usuario no volverá a introducir el código salvo que quiera cambiar de vivienda.

# 📄 Pantallas

---

# Pantalla 1 · Acceso

La primera vez que la aplicación se abra en un dispositivo se solicitará el código de acceso.

Elementos:

- Logo.
- Nombre de la aplicación.
- Campo de texto.
- Botón "Entrar".

---

# Pantalla 2 · Lista de la compra

Pantalla principal de la aplicación.

Mostrará:

- Nombre de la vivienda.
- Nombre de la lista.
- Productos pendientes.
- Productos comprados.
- Botón flotante para añadir productos.

---

Wireframe:

```text
┌────────────────────────────┐
│ 🏠 Casa Martínez           │
│ Lista principal            │
├────────────────────────────┤
│ ☐ Leche                    │
│ ☐ Pan                      │
│ ☑ Huevos                   │
│ ☐ Café                     │
│                            │
│                            │
│                        ＋   │
└────────────────────────────┘
```

---

# Pantalla 3 · Añadir producto

Se mostrará mediante un modal.

```text
┌────────────────────────────┐
│ Añadir producto            │
├────────────────────────────┤
│                            │
│ [____________________]     │
│                            │
│ Cancelar      Añadir       │
└────────────────────────────┘
```

---

# Pantalla 4 · Configuración

Permitirá:

- Cambiar de vivienda.
- Ver versión.
- Información de la aplicación.

No incluirá opciones avanzadas en la primera versión.

# Componentes

La interfaz se construirá reutilizando componentes.

---

## Button

Botón reutilizable.

Variantes:

- Primary
- Secondary
- Danger

---

## Modal

Utilizado para:

- Añadir producto.
- Confirmaciones.

---

## Header

Mostrará:

- Nombre de la vivienda.
- Nombre de la lista.

---

## ShoppingItem

Representa un producto.

Información:

- Nombre.
- Estado.

---

## Floating Action Button

Botón flotante para añadir productos.

Siempre visible.

---

## Input

Campo reutilizable.

Se utilizará para:

- Código de vivienda.
- Añadir productos.

---

# Estados de la interfaz

## Cargando

```text
Cargando lista...
```

---

## Lista vacía

```text
🛒

No hay productos pendientes.

¡Añade el primero!
```

---

## Error

```text
⚠️

No se ha podido cargar la información.
```

---

# Paleta de colores

Se utilizará una paleta sencilla.

## Primario

Verde.

Representa:

- Añadir.
- Confirmar.
- Producto comprado.

---

## Secundario

Azul.

Representa:

- Navegación.
- Información.

---

## Error

Rojo.

Representa:

- Eliminación.
- Errores.

---

## Fondo

Blanco o gris muy claro.

---

# Tipografía

Se utilizará una tipografía moderna y legible.

Preferiblemente:

- Inter
- Roboto

---

# Iconografía

Se utilizará una única librería de iconos.

Recomendación:

- Lucide React

---

# Responsive

La aplicación deberá adaptarse a:

## Móvil

Diseño principal.

---

## Tablet

Mayor separación entre elementos.

---

## Escritorio

La lista aparecerá centrada con un ancho máximo.

---

# Instalación como PWA

La aplicación podrá:

- Instalarse.
- Abrirse desde el icono del dispositivo.
- Funcionar como una aplicación nativa.

---

# Experiencia NFC

La experiencia ideal será:

```text
Escanear NFC

↓

La aplicación se abre

↓

La lista aparece inmediatamente

↓

Añadir producto

↓

Cerrar aplicación
```

Todo el proceso deberá durar menos de cinco segundos.

---

# Futuras mejoras

En próximas versiones podrán añadirse:

- Tema oscuro.
- Animaciones.
- Categorías.
- Favoritos.
- Gestos.
- Drag & Drop.
- Estadísticas.

---

# Conclusión

La interfaz priorizará la simplicidad y la rapidez sobre la cantidad de funcionalidades.

Cada decisión de diseño deberá responder a una única pregunta:

> ¿Hace que añadir un producto sea más rápido y sencillo?
